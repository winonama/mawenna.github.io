ECMAScript6 —— let 和 const

发表于 2017-09-01 | 分类于 技术 ， JavaScript

参考阮一峰ECMAScript6入门

文档地址http://es6.ruanyifeng.com/#docs/let

ES6 变量声明的六种方法

ES5 只有两种声明变量的方法：var命令和function命令。

ES6除了添加let和const命令，还有另外两种声明变量的方法：import命令和class命令。所以，ES6 一共有6种声明变量的方法。

let命令

基本用法

1. let声明的变量只在其所在的代码块中有效。即 块级作用域

块级作用域

ES5中只有全局作用域和函数作用域，在ES6中使用let和const增加了块级作用域。

在ES5中，内层变量可能会覆盖外层变量

块级作用域支持嵌套

块级作用域与函数声明

ES5 规定，函数只能在顶层作用域和函数作用域之中声明，不能在块级作用域声明。但浏览器没有遵守这个规定。

ES6 规定，块级作用域之中，函数声明语句的行为类似于let，在块级作用域之外不可引用。

浏览器兼容的块级函数声明类似于var

2. 不存在变量的提升。它所声明的变量需先声明，后使用。

3. 暂时性死区。如果区块中存在let和const命令，这个区块对这些命令声明的变量，从一开始就形成了封闭作用域，不再受外部的影响。凡是在声明之前就使用这些变量，就会报错。

存在全局变量tmp但块级作用域内使用let变量声明一个局部变量tmp，所以在let前进行赋值报错

4. 不允许重复声明。let不允许在相同的作用域内重复声明一个变量

for循环有一个特别之处，就是设置循环变量的那部分是一个父作用域，而循环体内部是一个单独的子作用域。

5. do表达式返回块级作用域的返回值

将作用域转为do表达式

const

基本用法

1. const声明一个只读的常量。一旦声明，常量的值就不能改变。

2. const一旦声明变量，就必须立即初始化，不能留到以后赋值。

3. 块级作用域：const的作用域与let命令相同：只在声明所在的块级作用域内有效。

4. 暂时性死区：const命令声明的常量也是不提升，同样存在暂时性死区，只能在声明的位置后面使用。

5. 不可重复声明：const声明的常量，也与let一样不可重复声明

本质

const实际上保证的，并不是变量的值不得改动，而是变量指向的那个内存地址不得改动。

但对于复合类型的数据（主要是对象和数组），变量指向的内存地址，保存的只是一个指针，const只能保证这个指针是固定的。

如果真的想将对象冻结，应该使用Object.freeze方法。

指向一个冻结的对象，添加新属性不起作用，严格模式时还会报错。

winow 顶层对象属性

ES6规定：var命令和function命令声明的全局变量，依旧是顶层对象的属性；另一方面规定，let命令、const命令、class命令声明的全局变量，不属于顶层对象的属性。
